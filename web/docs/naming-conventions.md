# Naming Conventions

## File Names

All component files use **kebab-case**.

```
app-layout.tsx
customer-list.tsx
data-table.tsx
bottom-nav.tsx
business-profile.tsx
```

## Component Names

Exported components inside those files use **PascalCase**.

```tsx
// file: customer-list.tsx
export function CustomerList() { ... }

// file: data-table.tsx
export function DataTable() { ... }

// file: bottom-nav.tsx
export function BottomNav() { ... }
```

## Imports

Import path uses kebab-case, imported symbol uses PascalCase — consistent with the file.

```tsx
import { CustomerList } from '@/pages/customers/customer-list'
import { DataTable } from '@/components/shared/data-table'
import { AppLayout } from '@/components/layout/app-layout'
```

## Why

- Kebab-case file names: consistent with URLs, avoids case-sensitivity bugs on Linux (case-insensitive on macOS, case-sensitive in production).
- PascalCase exports: React convention, required for JSX to treat them as components.

## Exceptions

- `index.ts`, `frontend.tsx`, `index.html` — entry points, keep as-is.
- `lib/utils.ts` — utility, not a component.
- `src/data/*.ts` — data files, not components, use kebab-case naturally.
- shadcn `ui/` components — generated by CLI, already kebab-case file names.
